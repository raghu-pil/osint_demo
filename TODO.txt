Goal: Given a URL / video / photo / audio, get maximum intel on the content
      AND the person who posted it.
================================================================================

Two parallel tracks on every investigation:
  CONTENT TRACK  — What is this? Is it real? Where did it originate?
  PERSON TRACK   — Who posted it? What else do we know about them?

--- NETWORK GRAPH (HIGH PRIORITY) ---

[ ] Deep graph expansion — friends/followers traversal to depth 4
    Current state: graph shows nodes from a single investigation (post → account
    → platforms found by Sherlock → media → identity pivots). It does NOT
    fetch the social network of the account being investigated.

    What needs to change:
    - Click any Account node → "Expand network" button appears
    - Fetch followers/following for that account (up to N, configurable)
    - Each fetched account becomes a new node; edges = "follows" / "followed by"
    - Recursively expandable up to depth 4 from the original subject
    - Depth 1: direct followers/following of subject
    - Depth 2: followers/following of those accounts
    - Depth 3–4: further out — reveals clusters, coordination patterns
    - Visual: node size = follower count, color = risk score, edge thickness = interaction frequency

    Implementation per platform:
    - Twitter/X: vxtwitter following endpoint + Nitter /followers page (no official API needed)
    - Instagram: unofficial graph API (fragile — wrap carefully)
    - Telegram: Telethon for public channel member lists
    - Reddit: /user/{u}/friends via JSON API

    Performance considerations:
    - Don't expand all at once — lazy load on click (expand one node at a time)
    - Cache fetched nodes in SQLite so re-opening the graph is instant
    - Add a "Stop expanding" button — depth 4 on a large account = thousands of nodes
    - Configurable max nodes per expansion (default: 50 per level)
    - Show "sampled 50 of 12,400 followers" when capped

    Why this is high priority:
    - Depth-4 graph expansion is the single most powerful feature for detecting
      coordinated inauthentic behaviour — clusters of accounts that follow each
      other but no one else, accounts created on the same day, identical posting
      patterns — all become visible only when you can see the network structure

[ ] Audio fingerprinting via AcoustID/Chromaprint
    - Identifies if audio clip is from a known song/broadcast
    - Useful for: detecting repurposed audio, fake "on-the-ground" recordings
      that are actually old news clips with new video overlaid
    - pip install pyacoustid + fpcalc binary

[ ] Speech detection and language identification
    - Use OpenAI Whisper (local, free, runs on CPU)
    - Detects: is there speech? what language? transcribe it
    - Transcription feeds into the existing NER pipeline (extract names,
      locations, phone numbers from spoken content)
    - Particularly valuable for WhatsApp voice notes circulating in India/Pak

[ ] Audio provenance red flags
    - Background noise analysis: indoor/outdoor/crowd (librosa)
    - Detect WhatsApp compression artifacts (specific bitrate/codec signature)
      — viral WhatsApp audio often has 3-4 generations of re-encoding
    - Detect abrupt cuts/splices (energy discontinuity detection)
    - Detect pitch shifting (common manipulation to disguise voice)

--- VIDEO (improve existing) ---

[ ] Extract and analyze audio track from video separately
    - Run full audio pipeline above on the extracted audio track
    - Many manipulated videos have authentic video + fake audio or vice versa

[ ] Replace Claude API calls with self-hosted Ollama (SOTA vision+text model)
    - Current: image analysis + post intelligence call Anthropic API
      (costs money per call, requires internet, sends content to third party)
    - Target: run a SOTA multimodal model locally via Ollama
      Recommended models (pick one based on available VRAM):
        * llava-llama3 or llava:34b  — strong vision, good text (16GB VRAM)
        * gemma3:27b                 — strong multimodal (16GB VRAM)
        * llama3.2-vision:90b        — best quality, needs 48GB+ VRAM
        * minicpm-v or moondream2    — lightweight, runs on CPU/8GB
    - Config: llm.provider = "ollama", llm.ollama_base_url already in config.yaml
    - Scope: backend/modules/image_analysis.py + backend/modules/post_analysis.py
    - Keep Anthropic as fallback if Ollama is not running
    - Benefit: no API cost, no data leaves the machine, faster on good GPU

[ ] Improve deepfake detection
    - Currently relies on Claude vision API (good but expensive per frame)
    - With Ollama in place: deepfake detection prompt runs locally at no cost
    - Add: FaceForensics++ style artifact detection (local, free)
